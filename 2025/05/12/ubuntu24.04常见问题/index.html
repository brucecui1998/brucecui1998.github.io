ubuntu界面登录黑屏及卡顿

问题背景

在 Ubuntu 24.04 上使用 NVIDIA 驱动 时，遇到了图形卡顿和偶尔登录黑屏的问题。默认情况下，Ubuntu 使用 Wayland，且桌面管理器为 GDM。
问题分析

DRM 模式设置缺失：

    初步分析发现，问题并非由 Wayland 与 NVIDIA 驱动的兼容性差 引起。通过撤销禁用 Wayland 的设置后，依然能够正常登录桌面，且没有出现黑屏问题。

启用 nvidia-drm.modeset=1：

    问题的核心原因在于 NVIDIA 驱动 与 DRM 模式设置 的配合。启用 nvidia-drm.modeset=1 内核参数后，问题明显得到解决。登录成功且性能不再卡顿。

解决步骤

启用 nvidia-drm.modeset=1：

    编辑文件 /etc/default/grub，在 GRUB_CMDLINE_LINUX_DEFAULT 中添加 nvidia-drm.modeset=1：

GRUB_CMDLINE_LINUX_DEFAULT=”quiet splash nvidia-drm.modeset=1”

更新 grub 配置并重启：

    sudo update-grub
    sudo reboot

不需要禁用 Wayland：

    经过测试，问题并非由于 Wayland 与 NVIDIA 驱动的兼容性差。即使在 Wayland 下，启用 nvidia-drm.modeset=1 也能解决卡顿和图形问题。

结果

启用 DRM 模式设置 (nvidia-drm.modeset=1) 后，登录成功且系统性能稳定，问题得以解决。

在禁用 Wayland 的情况下，问题依然可以解决，但实际上 Wayland 并非问题根源。

总结

DRM 模式设置 是解决 NVIDIA 驱动 在 Ubuntu 24.04 中图形卡顿和登录问题的关键。通过启用 nvidia-drm.modeset=1 参数，成功解决了图形问题。

禁用 Wayland 并非必须，但对于一些系统配置，禁用 Wayland 可能是一个可选的解决方案。

Note Ubuntu 24.04 SSH 修改端口不生效的问题记录

在 Ubuntu 24.04 中，当修改 /etc/ssh/sshd_config 中的 Port 配置并执行：

sudo systemctl restart ssh

后，发现 SSH 仍监听在旧端口（如 22），而非修改后的新端口。这是因为从 Ubuntu 22.10 开始，Ubuntu 引入了 OpenSSH 的 systemd socket 激活机制（ssh.socket），此机制会让 systemd 预先监听端口，然后按需激活 sshd 服务。

因此，sshd_config 中的 Port 设置将被忽略，因为监听行为由 systemd 控制。

官方说明链接：
🔗 https://discourse.ubuntu.com/t/sshd-now-uses-socket-based-activation-ubuntu-22-10-and-later/30189/13

该机制的引入主要是为了节省系统资源（如内存），但在实际运维过程中可能会导致行为与预期不符。
✅ 最简单的解决办法：禁用 socket 激活，恢复传统行为

  1. 停用并禁用 socket 激活机制
    sudo systemctl stop ssh.socket
    sudo systemctl disable ssh.socket

  2. 启用并重启传统的 sshd 服务
    sudo systemctl enable ssh.service
    sudo systemctl restart ssh.service

这样就可以重新让 sshd_config 中的 Port 配置生效，使用 sudo systemctl restart ssh 即可切换端口。
