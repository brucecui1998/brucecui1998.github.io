题目

给你一个按照非递减顺序排列的整数数组 nums，和一个目标值 target。请你找出给定目标值在数组中的开始位置和结束位置。

如果数组中不存在目标值 target，返回 [-1, -1]。

你必须设计并实现时间复杂度为 O(log n) 的算法解决此问题。

提示：

  • 0 <= nums.length <= 105
  • -109 <= nums[i] <= 109
  • nums 是一个非递减数组
  • -109 <= target <= 109

思路

与二分查找要求的条件（有序、无重复）不同的是，该题并没有说无重复元素。

寻找target在数组里的左右边界，有如下三种情况：

  • 情况一：target 在数组范围的右边或者左边，例如数组{3, 4, 5}，target为2或者数组{3, 4, 5},target为6，此时应该返回{-1, -1}
  • 情况二：target 在数组范围中，且数组中不存在target，例如数组{3,6,7},target为5，此时应该返回{-1, -1}
  • 情况三：target 在数组范围中，且数组中存在target，例如数组{3,6,7},target为6，此时应该返回{1, 1}

unet代码解析

我们首先根据论文中网络结构图来看unet的网络结构部分的代码，我们先来看UNet类中的__init__方法，

  • 首先是一个双卷积层self.down_conv用于收缩路径，对应图中的蓝色箭头

  • self.down_sample下采样层用于收缩路径，对应图中的红色箭头

  • self.middle_conv是图中底部的双卷积层，对应图中最底部的蓝色箭头

  • self.up_sample扩展路径的上采样层，经过上采样，特征数量减半，对应图中绿色箭头（注意不包含左边白色的）

  • self.up_conv用于扩展路径的双卷积层

  • self.concat裁剪并连接用于扩展路径的层对应灰色部分（注意这里才是左半边白色部分的512个特征通道）

  • self.final_conv最终经过1*1的卷积核输出把64个特征通道变成2个，也就是最后的388×388×2，其实这里就是一个二分类的操作，把图片分成背景和目标两个类别。

我们再来看unet中的forward方法，forward可以理解为沿着网络结构前向传播

  • pass_through变量，收集收缩路径的输出，以便稍后与扩展路径连接。也就是提前准备好灰色箭头的输入

  • 第一个for循环，可以理解为沿着U型结构的左边收缩路径走，len(self.down_conv)的长度为4，也就是双卷积层总共做4次，也和图中能对应上。

    • 首先是x = self.down_conv[i] (x) i是输入输出通道参数的索引，对应四个值，分别是(in_channels, 64), (64, 128), (128, 256), (256, 512)，x是输入的图片
    • pass_through.append(x)，接着收集一下收缩路径的输出
    • x = self.down_sample[i] (x) 最后池化层下采样

    以上三步操作重复执行四次，此时已经走到了U型结构的底部

  • 在底部，x = self.middle_conv(x)再来一个双卷积层，对应论文图中最底部的两个蓝色箭头

  • 第二个for循环，相同的道理，循环四次，分别先后进行上采样、拼接、反卷积

  • 最终x = self.final_conv(x)经过2个1*1的卷积核，就是一个二分类的操作，把图片分成背景和目标两个类别，并将x返回

以上是unet的网络结构。我们再看一下unet.py中的其他类class DoubleConvolution、class DownSample、class UpSample、class CropAndConcat其实就是刚才unet的forward中的具体操作

这里只讲解一个DoubleConvolution例子

在unet网络结构中的init方法的self.down_conv其实就是调用了自己编写的DoubleConvolution，那我们来看看双卷积层的操作是怎么定义的，首先在DoubleConvolution的init方法中，实际是通过torch来定义双卷积层的

  • nn.Conv2d
  • nn.ReLU

DoubleConvolution的forward方法其实就是将这些操作按顺序拼接起来,先卷积一下，再relu激活一下，再卷积一下，再relu激活一下，这样子就拼成了一个双卷积层的操作。class DownSample、class UpSample、class CropAndConcat与其原理类似，都是调用pytorch来定义网络结构

用std::vector构建二维数组在内存访问上性能不如arma::cube的原因有两个：

1、虽然一个vector向量内的地址连续，但是向量间内存地址不联系

tips大佬的话： 这里面机制有点复杂,EFI的引导检测机制是使用特殊的EFI分区中的efi程序,efi固件在进入时会扫描每个磁盘上的efi分区,把nvram里面保存的efi引导程序放在这里,你选一个就会根据记录找到对应的文件引导,但是老旧的legacy引导不是这样,让我们回到二十年前,那时候还没有efi固件,也不存在efi分区,引导方式就是简单的扫描零号扇区,因此只能通过选择设备区分不同的引导项,我说的很简单，其实细节远不止这样,你感兴趣可以看看GRUB的两阶段引导的具体实现,那个000H1就是保存在NVRAM里面的Windows引导项,它对应的efi程序是/EFI/Windows/bootmgr.efi.顺带再一提,虽然我们经验上说MBR对应Legacy引导，GPT对应UEFI引导,但是实际上它们直接不是绝对的对应关系,而是一个两两组合的二维矩阵,换句话说，MBR+EFI，GPT+Legacy也是可以的,虽然不被推荐,是Windows强行绑定起来,把人都惯成了思维定式
