Docker便捷搭建纯净的Linux环境

突然发现在docker下面可以很便捷的搭建纯净的Linux环境，删除的话也非常方便，干净，docker比虚拟机还是有不少好处的

Docker 和虚拟机（Virtual Machines，VMs）是两种不同的虚拟化技术，它们在多个方面有着重要的区别：

  1. 资源利用效率：
    • Docker：Docker 使用容器技术，容器共享宿主操作系统的内核，因此启动和运行的容器更加轻量级。容器之间共享操作系统内核，这使得容器更加节省资源，启动更快，占用的内存和磁盘空间较少。
    • 虚拟机：虚拟机是完整的虚拟操作系统，包括独立的内核。每个虚拟机需要分配一定数量的计算资源，包括内存和处理器，这使得虚拟机通常比较重量级，资源占用较高。
  2. 启动时间：
    • Docker：Docker 容器的启动时间通常在秒级别，非常快速。这使得容器适用于快速部署和伸缩。
    • 虚拟机：虚拟机的启动时间通常在分钟级别，相对较慢。虚拟机需要启动整个操作系统，因此启动时间较长。
  3. 部署一致性：
    • Docker：Docker 容器可以使用Docker镜像，其中包含了应用程序及其依赖项的快照，确保了应用程序的一致性和可移植性。
    • 虚拟机：虚拟机可以使用虚拟机镜像，但它们通常较大，不如Docker镜像轻便，因此在部署一致性方面可能不如Docker容器。
  4. 资源隔离：
    • Docker：Docker 使用 Linux 内核的命名空间和控制组来提供资源隔离。尽管容器之间共享操作系统内核，但它们仍然具有一定程度的资源隔离。
    • 虚拟机：虚拟机通过虚拟化层实现较高级别的资源隔离，每个虚拟机都有自己的独立内核和资源分配。
  5. 管理和维护：
    • Docker：Docker 容器的管理相对简单，因为容器可以轻松创建、删除和更新，而且可以使用 Docker Compose 或 Kubernetes 等工具进行集群管理。
    • 虚拟机：虚拟机的管理较复杂，因为它们需要管理操作系统的完整副本，包括升级、维护和备份。

总的来说，Docker 更适合容器化应用程序的部署和管理，特别是在需要快速部署和资源利用效率方面。虚拟机更适合运行不同操作系统的工作负载以及需要更严格的资源隔离的场景。选择使用哪种技术取决于你的具体需求和应用场景。在某些情况下，Docker 和虚拟机也可以结合使用，以充分利用它们各自的优势。

术语解释

  • 3 DoF

“3 DoF” 是 “3 Degrees of Freedom” 的缩写，中文通常翻译为“三自由度”。在机械工程和机器人技术中，自由度通常描述一个物体在空间中的移动和旋转的能力。对于3 DoF来说，它通常指的是以下三个方面的能力：

  1. 沿X轴旋转（俯仰，Pitch）
  2. 沿Y轴旋转（偏航，Yaw）
  3. 沿Z轴旋转（翻滚，Roll）

这意味着一个有3 DoF的设备或系统只能在这三个方向上进行旋转，而不能进行平移。

  • longitude, latitude, altitude
  1. **经度 (Longitude)**：是从地球的西半球到东半球的角度度量，它描述了位置相对于本初子午线（通过格林尼治的一条经线）的东西方向的位置。经度的范围是从-180°到+180°。其中，0°是本初子午线，正值表示东经，负值表示西经。
  2. **纬度 (Latitude)**：是从地球的南极到北极的角度度量，它描述了位置相对于赤道的南北方向的位置。纬度的范围是从-90°到+90°。其中，0°是赤道，正值表示北纬，负值表示南纬。
  3. **海拔高度 (Altitude)**：描述了一个位置相对于平均海平面的高度。通常，海拔高度是从地球的平均海平面开始测量的，但在飞行等其它领域中，可能会使用其它参考点，如机场的跑道等。所以，是的，当我们通常提到”Altitude”时，我们是指海拔高度。但在某些上下文中，它也可能指其他高度测量方式。

综上所述，通过这三个坐标，我们可以精确地描述地球上任何位置的位置和高度。

常用坐标系

  • 地理坐标系

经纬度和高度组成的球面坐标系，一般顺序为经度( λ )(单位rad)，纬度(L)(rad)，高度(h)(m) 组成。也叫LLA.

是使用三维球面来定义地球表面位置，以实现通过 经纬度 对地球表面点位引用的坐标系。

一个地理坐标系包括 角度测量单位、本初子午线和参考椭球体三部分。在球面系统中，水平线是等纬度线或纬线。垂直线是等经度线或经线。地理坐标系依据其所选用的本初子午线、参考椭球的不同而略有区别。地理坐标系可以确定地球上任何一点的位置。首先将地球抽象成一个规则的逼近原始自然地球表面的椭球体，称为参考椭球体，然后在参考椭球体上定义一系列的经线和纬线构成经纬网，从而达到通过经纬度来描述地表点位的目的。需要说明的是经纬地理坐标系不是平面坐标系，因为度不是标准的长度单位，不可用其直接量测面积长度

  • 局部导航坐标系

也叫当地导航坐标系(local navigation frame) ，当地水平导航坐标、测地坐标系、地理坐标系或地面点坐标系。

从定义来分类，局部切线平面可分为基于垂直和水平尺寸定义的平面，其表现在纵坐标为上还是下。纵坐标为上时，称为ENU（东、北、天）坐标系，主要用于地理方面；纵坐标为下时，称为NED（北、东、地）坐标系，特别用于航空航天。

  • 地心惯性坐标系(ECI, I系)

地心惯性坐标系用XYZ表示，原点为地球中心，X轴指向春分点，既黄道平面(地球绕太阳公转的平面)与赤道的交点。

  • 地心地固坐标系(ECEF, 或者叫地球坐标系， E系)

原点在地心，X指向0经度0纬度点(本初子午线与赤道交点)。

总结：ECI坐标系为绝对惯性系，静止不动，ECEF随着地球一起转动。
