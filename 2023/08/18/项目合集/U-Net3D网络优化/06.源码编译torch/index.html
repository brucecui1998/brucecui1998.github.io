06.源码编译torch

Debian源码编译torch-v1.13.1

本机环境：

  • Debian 12.1
  • gcc/g++ 12.2
  • python 3.10.12

安装过程参考：https://github.com/pytorch/pytorch/tree/v1.13.1#from-source

准备工作

准备cuda环境

参考本博客 “杂项汇总”-“搭建科研环境远程服务器”

准备cudnn

下载cudnn （cudnn Support Matrix版本对照）

安装cudnn

准备anaconda环境

Install Dependencies

准备源码以及第三方库

由于网速问题，torch源码的submodule很难clone下来，这里我们先手动把torch源码所需要的submodule全部下载下来拷在u盘准备拷到linux系统上。

submodule下载可以看一下.gitsubmodule

编译源码

将pytorch-1.13.1.zip和这些submodule都上传到linux服务器。

编译

常见报错

  • 报错：C++: fatal error: Killed signal terminated program cc1plus

https://blog.csdn.net/weixin_44796670/article/details/121234446

  • 编译报错error: invalid conversion from ‘void’ to ‘const char’ [-fpermissive]

在CMakeLists.txt中添加

  • 报错：cc1plus: warnings being treated as errors

将CMakeLists.txt中有关-Werror=return_type的设置给注释掉

曙光x86平台源码编译torch-v1.10.0

  • Python 3.6.2 or later
  • C++14 compiler
  • AMD ROCm 4.0 and above installation
